Introducere

ECAW este o aplicatie web extensibila pentru crearea digitala a unor ilustrate de diverse categorii si impartasirea acestora cu cei dragi. Aplicatia este una de tip SPA, deoarece toate codurile necesare - HTML , JavaScript și CSS - sunt preluate cu o singură pagină de încărcare.

Tehnologii

HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afișate într-un browser (sau navigator). Va fi folosit in cadrul scheletului principal al proiectului.

CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atașa elementelor HTML prin intermediul unor fișiere externe sau în cadrul documentului. CSS se poate utiliza și pentru formatarea elementelor XHTML, XML și SVGL. Intregul design al aplicatiei va fi realizat si formatat prin intermediul CSS.

Javascript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalități în paginile web, codul Javascript din aceste pagini fiind rulat de către browser.

Node.js

Node.js este o platformă de dezvoltare a surselor deschise pentru executarea codului JavaScript din partea serverului. Node este util pentru dezvoltarea aplicațiilor care necesită o conexiune persistentă de la browser la server și este adesea utilizat pentru aplicații în timp real, cum ar fi chat-uri, fluxuri de știri și notificări push web.

GitHub

Reprezintă un sistem de control al versiunilor codului sursă care va fi folosit pentru o mai bună administrare cât și pentru lucrul în echipa.

MongoDb

Reprezintă o bază de date NoSQL orientată către documente, folosită pentru stocarea datelor cu volum mare. Se încadrează în categoria bazelor de date de tip NoSQL.

Arhitectura aplicației

Utilizând tehnologiile menționate în secțiunea anterioară, aplicația va fi structurată în mai multe module, după cum urmează:

1. Login

Formularul de LOGIN se afla pe prima pagina a aplicatie. Utilizatorul introduce datele personale: username-ul si parola iar apoi daca datele sunt introduse corect va fi redirectionat catre pagina principala a aplicatiei.

Din punct de vedere al implementarii, username-ul si parola sunt trimise catre Server in body. Backend-ul proceseaza datele, cauta in baza de date in tabela USERS unsername-ul si parola primite din body. Parola fiind criptata cu Bcrypt, se va compara daca parola din body corespunde cu parola decriptata aflata in baza de date.

Daca in urma verificarii, datele corespund, utilizatorul va fi trimis carte pagina principala a aplicatiei, iar serverul va trimite jwt pe care clientul il va salva in cookie din motive de securitate, pentru a nu putea fi accesat de orice script aflat in pagina; altfel conectarea va esua iar el va fi instiintat printr-un mesaj in partea de sus a paginii. Pe partea de backend, serverul va trimite un status code si un mesaj descrise in functie de caz in Swaggerul atasat. (TO DO:)

2. Register

Formularul de REGISTER poate fi accesat prin apasarea butonului Register aflat in formularul de "Login". Utilizatorul va introduce username-ul dorit iar mai apoi va trebui sa introduca parola de doua ori. Server-ul va receptiona prin body datele utilizatorului, criptand parola cu ajutorul pachetului Bcrypt, iar mai apoi va salva datele in tabela USERS.

Pentru cazurile de eroare, serverul va trimite status cod-urile descrise in swagger-ul atasat, iar Utilizatorul va fi instiintat printr-un mesaj care va aparea in partea de sus a paginii. De exemplu pentru cazul in care username-ul se afla deja in baza de date, Server-ul va trimite catre Client status code-ul 409 impreuna cu mesajul: "Username already exists". Iar Clientul va afisa mesajul "Username already exists" pentru ca Utilizatorul sa fie instiintat de ce nu s-a finalizat cu succes inregistrarea.

3. Pagina principală

  • In interiorul paginii principale putem regasi 3 sectiuni.
    1. Prima sectiune, cea aflata in stanga paginii este destinata pentru istoricul cardurilor. Aceste fiind ordonate de sus in jos dupa data creearii.
    2. In centru se regaseste sectiunea in care se afla canvasul.
    3. In partea din dreapta se alfa toolbar-ul pentru editarea cardului, secriunea de Search unde se pot cauta diverse imagini, iar mai jos, in functie de modul de creare sau editare, se pot regasi butoanele de Save, New card si Share.
    1. Crearea unui card nou

      Imediat dupa login utilizatorul esre redirectionat catre pagina principala unde poate sa creeze un nou card dorit. Pentru inceput, clientul poate folosi toolbar-ul aflat in partea din dreapta a paginii. Fiecare tool reprezinta un buton care randeaza, in functie de cel selectat, obictul dorit, pentru implementare s-au folosit clasele

      • Rect
      • Line
      • Textbox
      • Circle

      Pentru butonul de Draw s-a folosit modul "isDrawingMode" pe "true" daca acesta doreste sa fie in acest mod sau "false" atunci cand Utilizarorul doreste sa iasa din acest mod. Pentru interactiunea cu utilizatorul, butonul va ramane selectat pe tot parucursul modului de desenare.

      Cu ajutorul color picker-ului se poate selecta culoarea dorita. Pe langa asta se mai poate select si dimensiunea liniei.

    2. Sectiunea "Search" este resprezentata print-un input si un buton in interiorul lui pentru a finaliza cautarea. Clientul face un POST catre Server, astfel evitand eroare de CORS, cu continutul dupa care se doreste sa se faca cautarea. Serverul receptioneaza continutul prin intermediul body-ului, face un https get catre API-ul extern: https://pixabay.com/ punand in interiorul URL-ului o cheie specifica si continutul. Acest https get va raspunde cu un array de forma: TO DO, in cazul in care se reuseste conectarea la API, altfel va returna o eroare de tipul "Request failed" sau "Invalid content-type".
    3. Ultima sectiune, se regasesc butoanele Save, New card si Share
        Save

        Daca utilizatorul nu se afla in modul de editare, clientul, prin butonul Save, va trimite catre Server prin metoda POSTpe ruta "card/create" jwt-ul pe care il ia din cookie prin metoda getCookie, img pe care o salveaza in imagine prin metoda toDataUrl si data care reprezinta intregul continut al calvasului sub forma de JSON string. Serverul intercepreaza datele si le salveaza in tabela CARD. Toate raspunsurile se regasesc in swagger iar serverul raspunde cu toate cazurle enuntate.
