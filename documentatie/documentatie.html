Introducere

ECAW este o aplicatie web extensibila pentru crearea digitala a unor ilustrate de diverse categorii si impartasirea acestora cu cei dragi. Aplicatia este una de tip SPA, deoarece toate codurile necesare - HTML , JavaScript și CSS - sunt preluate cu o singură pagină de încărcare

Tehnologii

HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afișate într-un browser (sau navigator). Va fi folosit in cadrul scheletului principal al proiectului.

CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atașa elementelor HTML prin intermediul unor fișiere externe sau în cadrul documentului. CSS se poate utiliza și pentru formatarea elementelor XHTML, XML și SVGL. Intregul design al aplicatiei va fi realizat si formatat prin intermediul CSS.

Javascript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalități în paginile web, codul Javascript din aceste pagini fiind rulat de către browser.

Node.js

Node.js este o platformă de dezvoltare a surselor deschise pentru executarea codului JavaScript din partea serverului. Node este util pentru dezvoltarea aplicațiilor care necesită o conexiune persistentă de la browser la server și este adesea utilizat pentru aplicații în timp real, cum ar fi chat-uri, fluxuri de știri și notificări push web.

GitHub

Reprezintă un sistem de control al versiunilor codului sursă care va fi folosit pentru o mai bună administrare cât și pentru lucrul în echipa.

MongoDb

Reprezintă o bază de date NoSQL orientată către documente, folosită pentru stocarea datelor cu volum mare. Se încadrează în categoria bazelor de date de tip NoSQL.

Arhitectura aplicației

Utilizând tehnologiile menționate în secțiunea anterioară, aplicația va fi structurată în mai multe module, după cum urmează:

1. Register

Formularul de înregistrare va consta în înregistrarea cu ajutorul contrului de Facebook. Prin intermediul protocolului Oauth2 vom reține datele acestuia. După apăsarea butonului de REGISTER serverul va salva body-ul în tabela USERS. Dacă înregistrarea va fi cu succes, serverul va trimite statusul 201, iar clientul va încarca pagina principală. În cazul în care Înregistrarea nu se efectuează cu succes serverul va returna 400/500.

2. Login

Pentru a putea interacționa cu aplicația, fiecare utilizator trebuie să se logheze cu ajutorul contului de Facebook. Clientul trimite către server contul de facebook al utilizatorului. În cazul în care datele sunt validate cu succes în tabela USERS, serverul trimite către client un răspuns cu statusul 200, și îl redirecteaza catre pagina principala; altfel va trimite statusul 404 și va afișa un mesaj de eroare.

3. Pagina principală

După logare, utilizatorul va fi redirecționat către pagina principală care conține spațiul pentru realizarea cardului dorit. Acest spațiu este un canvas în care utilizatorul poate uploada fișiere din computerul personal sau poate cauta prin intermediul câmpului de Search aflat în dreapta sus, orice imagine dorește; poate modifică elementele atașate prin drag & drop sau editarea cu ajutorului toolbar-ului aflat în dreapta paginii.

În stânga paginii, utilizatorul poate accesa cardurile pe care le-a realizat în trecut.

Când utilizatorul dorește să caute o imagine cu ajutorul câmpului SEARCH, serverul va caută cu ajutorul unui API bazat pe google search (serpapi.com) și va trimite către client imaginile găsite, iar clientul va putea selecta o imagine din listă.

Atunci când utilizatorul apasă butonul de Textbox, acesta poate introduce textul dorit. De asemenea, sub butonul de SEARCH se va găsi toolbar-ul prin care utilizatorul poate edita după cum dorește elementele introduse în canvas. La apăsarea butonului SAVE aflat sub canvas, clientul trimite toate datele aflate pe card. Serverul primește prin body-ul requestului infomațiile pe care mai apoi le salvează în tabelele corespunzatore.

Dacă salvarea se va finaliza cu succes serverul trimite către client statusul 201 și mesajul "Record has been inserted successfully". Dacă nu va trimite statusul 500.

În cazul în care utilizatorul dorește să aibă un card privat, prin apăsarea butonului PRIVATE se va seta un câmp în baza de date.

Aplicația suporta și 3 comenzi rapide pentru manipularea obiectelor:

  • ctrl c: copiere
  • ctrl x: tăiere
  • ctrl v: lipire

Acest lucru se realizeaza prin înregistrarea unui eveniment care ascultă la onkeyup și verifică dacă evenimentul are codul specific.

4. Link-ul către card

Link-ul care poate fi partajat prin rețelele de socializare va putea fi accesat de alți utilizatori. Serverul face un get pe baza id-ului din url iar clientul afișează cardul.

5. Formularul de editare

Formularul de editare poate fi accesat doar de cei ce au făcut cardul selectat. Utilizatorul selectează cardul pe care dorește să îl editeze. În secțiunea canvasului se va randa cardul iar apoi se va putea modifică în mod obișnuit cu toolbar-ul aflat în dreapta paginii. La apăsarea butonului SAVE se va face update cu noile modificări în tabele prin metodă PUT. Serverul caută cardul seletat pe baza id-ului aflat în parametrii requestului iar mai apoi va lua din request câmpurile modificate și va face update în tabelele corespunzătoare. Dacă operațiunea va fi finalizată cu succes, serverul va trimite status-ul 200 și mesajul "Record has been updated successfully". Altfel va întoarce status-ul 500.

Concluzie

În concluzie, aplicația ECAW este o aplicație Web extensibilă pentru crearea digitală a unor ilustrate de diverse categorii și împărtășirea acestora cu cei dragi. Pentru editare, vor fi implementate operații precum undo-redo, drag & drop etc. Ilustrată virtuală va poate fi partajată pe diverse rețele sociale și prin poștă electronică.
