Introducere

Ecaw este o aplicatie web extensibila pentru crearea digitala a unor ilustrate de diverse categorii si impartasirea acestora cu cei dragi.

Tehnologii

HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afișate într-un browser (sau navigator). Scopul HTML este mai degrabă prezentarea informațiilor – paragrafe, fonturi, tabele ș.a.m.d. – decât descrierea semanticii documentului.HTML este prescurtarea de la Hyper Text Mark-up Language si este codul care sta la baza paginilor web. Va fi folosit in cadrul scheletului principal al proiectului .

CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atașa elementelor HTML prin intermediul unor fișiere externe sau în cadrul documentului. CSS se poate utiliza și pentru formatarea elementelor XHTML, XML și SVGL. Intregul design al aplicatiei se va fi realizat si formatat prin intermediul CSS.

Javascript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalități în paginile web, codul Javascript din aceste pagini fiind rulat de către browser.

Node.js

Node.js este o platformă de dezvoltare a surselor deschise pentru executarea codului JavaScript din partea serverului. Node este util pentru dezvoltarea aplicațiilor care necesită o conexiune persistentă de la browser la server și este adesea utilizat pentru aplicații în timp real, cum ar fi chat-uri, fluxuri de știri și notificări push web.

GitHub

Reprezinta un sistem de control al versiunilor codului sursa care va fi folosit pentru o mai buna administrare cat si pentru lucrul in echipa.

MongoDb

Reprezinta un o baza de date NoSQL orientata catre documente, folosita pentru un pentru stocarea datelor cu volum mare. Se incadreaza in categoria unei baze de date NoSQl.

Arhitectura aplicatiei

Utilizand tehnologiile mentionate in sectiunea anterioara, aplicatia va fi structurata in mai multe module, dupa cum urmeaza:

1. Login

Petru a putea interactiona cu aplicatia, fiecare utilizator trebuie sa se logeze cu ajutorul contului de Facebook. Clientul trimite catre server contul de facebook al utilizatorului. In cazul in care datele sunt stocate cu succes in tabela USERS, serverul trimite catre client un raspuns cu statusul 200 si cu messajul: Login Successful, altel statusul 404.

2. Pagina principala

Dupa logare, utilizatorul va fi redirectionat catre pagina principala care contine spatiul pentru realizarea cardului dorit. Acest spatiu este un canvas in care utilizatorul poate uploada fisiere din computerul personal sau poate cauta prin intermediul campului de Search aflat in dreapta sus, orice imagine doreste; poate modifica elementele atasate prin drag & drop sau editarea cu ajutorului toolbar-ului aflat in dreapta paginii. Cand utilizatorul doreste sa caute o imagine cu ajutorul campului SEARCH, serverul va cauta cu ajutorul lui google search api (serpapi.com) si va trimite catre client imaginile gasite, iar clientul va putea selecta o imagine din rezultatele gasite. Atunci cand utilizatorul apasa butonul de Textbox, acesta poate introduce textul dorit, de asemenea, sub butonul de SEARCH se va gasit toolbar-ul prin care utilizatorul poate edita dupa cum doreste elementele introduse in canvas. La apasarea butornul SAVE aflat sub canvas, clientul trimite toate datele aflate pe card. Serverul primeste prin body-ul requestului infomatiile pe care mai apoi le salveaza in tabelele corespunzatore. Daca salvarea se va finaliza cu succes serverul trimite catre client statusul 200 si mesajul: Insert successful. Daca nu va trimite statusul 404.

3. Link-ul catre card

Link-ul care poate fi partajat prin reletele de socializare va putea fi accesat de cei care doresc. Serverul face un get pe baza id-ului din url iar clientul afiseaza cardul.

4. Formularul de editare

Formularul de editare poate fi accesat doar de cei ce au facut cardul selectat. Utilizatorul selecteaza cardul pe care il doreste sa il editeze. In sectiunea canvasului se va randa cardul iar apoi se va putea modifica in mod obisnuit cu toolbar-ul aflat in dreapta paginii. La apasarea butornului "SAVE" se va face update cu noile modificari in tabele prim metoda put. Serverul cauta cardul seletat pe baza id-ului aflat in parametrii requestului iar mai apoi va lua din request campurile modificate si va face update in tabelele corespunzatoare. Daca operatiunea va fi finalizata cu succes, serverul va trimite statusul: 201 si mesajul Update Successful. Altfel va intoarce status code-ul : 404.

Concluzie

În concluzie, aplicatia ECAW este Web extensibila pentru crearea digitala a unor ilustrate de diverse categorii si impartasirea acestora cu cei dragi. Pentru editare, vor fi implementate operatii precum undo-redo, drag & drop etc. Ilustrata virtuala va poate fi partajata pe diverse retele sociale si prin posta electronica.
